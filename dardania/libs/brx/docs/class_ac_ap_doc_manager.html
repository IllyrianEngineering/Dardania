◆ curDocument()

AcApDocument * AcApDocManager::curDocument ( ) const
pure virtual

Returns current document.

Current document may be different from the active document (drawing). You may change current document using AcApDocManagerImp::setCurDocument method. All document dependent functions (like workingDatabase() for example) work with current document. Typically current document is the same as active document (drawing), unless explicitly set otherwise.

Returns
current document pointer

◆ setCurDocument()

Acad::ErrorStatus AcApDocManager::setCurDocument ( AcApDocument *  pDocument,
AcAp::DocLockMode  lockMode = AcAp::kNone,
bool  makeActive = false 
)
pure virtual

Sets the passed document to become current document.

This function allows external application to set a given document to become temporarily current document without the need to actually activating it (switching MDI drawing windows). Remember to switch back documents. Failing to do so may result in an application crash.

Parameters
[in]pDocumentbecomes current document
[in]lockModelock mode to set on the document
[in]makeActiveif true document becomes also MDI active
Returns
Acad::eOk if successful,
Acad::eNoDocument if passed NULL document pointer.
Remarks
Passing true for makeActive has the same effect as calling AcApDocManager::activateDocument
