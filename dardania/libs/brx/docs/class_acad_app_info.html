Writes a pair of 'global' and 'localised' command names to the AcadAppInfo object's Registry data.

Parameters

[in]localisedCommand'localised' command name (used as data for the value key)
Returns

◆ writeGroupNameToRegistry()

UX_EXPORT_FUNCTION AcadApp::ErrorStatus AcadAppInfo::writeGroupNameToRegistry ( const ACHAR *  szGroupName)

Writes a logical 'Command Group' name to the Windows Registry data for this AcadAppInfo object.

Parameters
[in]groupNamethe name of the command group, which is associated with this Application
Returns
eOk if successful, eInvalidValue or eInvalidKey or eRegAccessDenied else; eRejected is returned if no call to AcadAppInfo::writeToRegistry() was made before; eUnknownError is returned for other registry errors
Remarks
when a 'CommandGroup' is defined in Registry, all commands defined by the Application can be called either by '_globalCommand' or 'localisedCommand', as well as using 'commandGroup._globalCommand' or 'commandGroup.localisedCommand' syntax;
  • multiple calls to this function will create multiple group entries;
  • a prior call to AcadAppInfo::writeToRegistry() function is nessecary to determine whether the ObjectDRX or BRX part of the CAD system's registry is used;

◆ writeToRegistry()

UX_EXPORT_FUNCTION AcadApp::ErrorStatus AcadAppInfo::writeToRegistry ( bool  isDRX = false,
bool  onlyUseHKCU = false 
)

Writes the AcadAppInfo object's informations into Windows Registry.

Parameters
[in]isDRXdetermines whether the informations are written into ObjectDRX or BRX section
[in]onlyUseHKCUif false, an attempt to write to HKEY_LOCAL_MACHINE is made first; if true, all informations are written to HKEY_CURRENT_USER immediately
Returns
eOk if successful, eInvalidValue or eInvalidKey or eRegAccessDenied else
Remarks
if there is no write-access to HKEY_LOCAL_MACHINE section, all informations are automatically written to HKEY_CURRENT_USER section, so informations are never lost
