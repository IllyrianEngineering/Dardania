◆ close()

UX_EXPORT_FUNCTION Acad::ErrorStatus AcDbObject::close ( )

Closes the object.

If the object is database resident, all changes are done. All appropriate notifications are sent. If the object is already notifying, the function does not close again. If the object is NOT database resident, call delete instead of close, otherwise your application is causing a memory leak. On database resident objects, never call delete, always use close.

Returns
Acad::eNoDatabase, if object is not database resident
Acad::eCloseWasNotifying, if the object is already notifying
return code of subclose() if this subfunction does not return Acad::eOk
Acad::eOk, else
