◆ atEndOfObject()

bool AcDbDxfFiler::atEndOfObject ( )
virtual

The default implementation of this method is to terminate AutoCAD.

The implementation of this function should be that if there is no more of the object's information to read, then set the filer's internal error status to Acad::eOk, thus allowing it to move on and return true.

◆ atEOF()

bool AcDbDxfFiler::atEOF ( )
virtual

The default implementation of this method is to terminate AutoCAD.

It should return true if the filer is either at a subclass data marker in the object or at the start of the object's xdata or at the end of the object's data.

◆ atExtendedData()

bool AcDbDxfFiler::atExtendedData ( )

◆ precision()

int AcDbDxfFiler::precision ( ) const
virtual

The default implementation of this method is to return AcDb::kDfltPrec.

For the ObjectARX internal filer that handles DXF file I/O, this function always returns AcDb::kDfltPrec (a value of -1) when filing in. When filing out, if the filer's setPrecision() method (see AcDbDxfFiler::setPrecision()) has not been called, then AcDb::kDfltPrec is returned, otherwise the precision set by the last call to setPrecision() is returned.

virtual

◆ writeXDataStart()

Acad::ErrorStatus AcDbDxfFiler::writeXDataStart ( )
virtual

If the filer is filing to AutoLISP or ADSRX, then this function writes out the AcDb::DxfCode::kDxfXDataStart to indicate the start of xdata.

If the filer is filing to a DXF file, then this function does nothing.
